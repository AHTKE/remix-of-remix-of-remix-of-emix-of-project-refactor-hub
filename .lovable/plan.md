## نطاق الطلب
طلبت إعادة هيكلة كاملة للتطبيق:
- صفحة رئيسية فيها 3 أزرار (طالب، دعم فني، معلم)
- نظام تسجيل/دخول طالب كامل + لوحة طالب فيها 9 خيارات
- لوحة دعم فني (تسجيل دخول بكود، تأكيد دفع، حظر، دردشة، تقارير درجات)
- لوحة معلم (رفع حصص، اشتراكات، كتب، رسائل، حظر، خلفية التطبيق…)
- نظام امتحانات + شرط اجتياز 50% + 3 محاولات
- محفظة + وسائل دفع + ربط مع بوت تيليجرام للطرفين
- وضع ليلي/نهاري + زر رجوع في كل صفحة + إشعارات

## الوضع الحالي
المشروع فعلاً فيه بنية كبيرة شغّالة:
- مسارات للطالب (`student.*`)، للأدمن (`admin.*`)، webhook تيليجرام (`api/public/telegram/webhook.ts`)
- ملفات backend: `bot-handler.server.ts`, `bot-features.server.ts`, `repo.server.ts`, `telegram.server.ts`
- صفحة بدء/login موجودة لكن بطابع مختلف عن اللي طلبته

## المشكلة
لو نفّذت كل اللي فوق دفعة واحدة:
1. هكسر الكود الموجود اللي شغّال مع البوت
2. الرد هيكون مئات الملفات في نوبة واحدة بدون اختبار حقيقي
3. غالبا حاجات كتير مش هتشتغل

الأفضل أقسّمها على مراحل، وكل مرحلة نتأكد إنها شغّالة قبل اللي بعدها.

## الخطة المقترحة (مراحل)

### المرحلة 1 — الصفحة الرئيسية الجديدة + التنقل
- إعادة كتابة `src/routes/index.tsx`: 3 أزرار كبيرة (👨‍🎓 طالب، 🧑‍🔧 دعم فني، 👨‍🏫 مدرس) فوق خلفية
- زر تبديل الوضع الليلي/النهاري في الهيدر
- زر رجوع موحّد (component) يُستخدم في كل الصفحات الداخلية
- إصلاح عرض الكورسات عند الطالب (`student.courses.tsx`) — لازم أفحص ليه مش شغال

### المرحلة 2 — مسار الطالب
- صفحة "واجهة الطالب" (`/student-portal`) فيها زرّي: إنشاء حساب / لدي حساب
- نموذج تسجيل (اسم رباعي، رقم الطالب، رقم ولي الأمر، كلمة مرور + تأكيد، الصف)
- نموذج دخول (اسم رباعي + كلمة مرور)
- حفظ الجلسة محلياً + Cookie آمن، تحويل للوحة الطالب بعد النجاح
- لوحة الطالب: 9 بطاقات (الحصص المدفوعة، الاشتراكات، الكتب، الدعم، المحفظة، سجل النشاط، الإشعارات، المكافآت، خروج)

### المرحلة 3 — منطق الحصص والامتحانات
- صفحة الحصص: عرض الحصص المتاحة بسعرها، شراء من المحفظة
- بعد الشراء: فيديو + PDF شرح + PDF واجب + PDF حل
- نظام الامتحان: MCQ، تصحيح تلقائي، شرط 50%، أقصى 3 محاولات
- قفل الحصة التالية حتى اجتياز الامتحان
- نظام النقاط/المكافآت بعد كل اختبار

### المرحلة 4 — الاشتراكات والكتب
- صفحة الاشتراكات: قائمة، شراء بالمحفظة، عرض الحصص المضمنة
- صفحة الكتب: قائمة، نموذج طلب (اسم، هاتف، عنوان، أقرب مكتبة)، خصم الثمن، انتظار رد الدعم

### المرحلة 5 — المحفظة والدعم
- صفحة المحفظة: إدخال مبلغ، اختيار وسيلة دفع، تفاصيل التحويل، رفع صورة الإيصال
- صفحة دعم الطالب: رسائل + رفع صور + ردود الدعم + كلمات مفتاحية تلقائية
- صفحة الإشعارات: عدّاد + قائمة + رد على الدعم فقط

### المرحلة 6 — لوحة الدعم الفني
- تسجيل دخول الدعم (اسم + كود يحدّدهم المعلم)
- تأكيد/رفض عمليات الدفع
- حظر/رفع حظر الطلاب
- لوحة عمليات الطلاب (من له رصيد، كيف صُرف)
- دردشة + ترتيب درجات الاختبارات حسب الصف

### المرحلة 7 — لوحة المعلم
- تسجيل دخول صارم: الاسم = `Mahmoud only`، الكود = `HHDV/58HR`، الرقم = `01050747978`
- رفع حصص (الحقول كاملة + أسئلة)
- إدارة اشتراكات، كتب، وسائل دفع (مع رمز سري للحذف)
- مراقبة الدعم (إضافة/حذف/سجل دخول)
- عرض عمليات التحويل المؤكدة + الإجمالي
- رسائل عامة (لصف أو للكل، مدة ظهور)
- ردود على رسائل الطلاب (نص/صوت/صورة)
- حظر طلاب + تعديل خلفية التطبيق

### المرحلة 8 — تكامل البوت
- مراجعة `bot-handler.server.ts` للتأكد من إرسال البيانات لكل طرف:
  - الطالب يستقبل: ردود الدعم/المعلم، تأكيد الشحن، إشعارات الحصص
  - الدعم يستقبل: طلبات الشحن، رسائل الطلاب
  - المعلم يستقبل: رسائل الطلاب، تقارير
- تخزين الميديا في `TELEGRAM_MEDIA_CHANNEL_ID` والبيانات في `TELEGRAM_DATA_CHANNEL_ID`

## ملاحظات تقنية
- Storage: المشروع شغّال بدون DB خارجي عبر قنوات تيليجرام (`repo.server.ts`). هكمّل بنفس النمط.
- Auth: Cookies + `SESSION_SECRET` الموجود
- لا أقدر أختبر فعلياً مع البوت بدون التأكد إن الـ secrets اللي حطيتها صحيحة وإن الـ webhook مسجّل
